Frequently Asked Questions

❓ How do I enable file URL access for local markdown files?
  1. Open Chrome and go to chrome://extensions/
  2. Find MarkView in the list
  3. Click "Details"
  4. Scroll down to "Allow access to file URLs"
  5. Toggle the switch to ON

Now you can open local .md files directly in Chrome!

❓ Why isn't the extension working on GitHub blob pages?

MarkView is designed to work with raw markdown files, not GitHub's rendered HTML pages.

Solution: Click the "Raw" button on the GitHub page to view the raw markdown file. MarkView will automatically render it beautifully.

Example:

  • ❌ Won't work: github.com/user/repo/blob/main/README.md
  • ✅ Will work: raw.githubusercontent.com/user/repo/main/README.md
❓ How do I change the theme to dark mode?
  1. Click the MarkView extension icon in Chrome toolbar
  2. In the popup, find the "Theme" dropdown
  3. Select "Dark" for dark mode, "Light" for light mode, or "Auto" to follow your system preference

You can also use the keyboard shortcut: Ctrl+Shift+T (or Cmd+Shift+T on Mac)

❓ Can I customize the font and layout?

Yes! MarkView offers extensive customization:

  • Font Family: Choose from 9 fonts (Inter, Merriweather, Roboto, etc.)
  • Font Size: Adjust with the slider (12px - 24px)
  • Line Height: Control spacing between lines (1.0 - 2.5)
  • Max Width: Set content width (600px - 1200px)
  • Centered Layout: Toggle centered layout on/off

All settings are accessible from the extension popup.

❓ Does auto-refresh work with local files?

No, auto-refresh only works with external files (HTTP/HTTPS URLs) due to Chrome security restrictions.

For local files: You'll need to manually refresh the page (F5) after editing the markdown file.

For external files: Enable auto-refresh in the popup settings, and the page will update automatically every 2 seconds.

❓ How do I view the table of contents?

The Table of Contents (TOC) sidebar appears automatically for markdown files with headings.

Show/Hide TOC:

  • Click the TOC toggle button (≡ icon) in the top-right corner
  • Or use keyboard shortcut: Ctrl+Shift+O (or Cmd+Shift+O on Mac)

The TOC automatically highlights the current section as you scroll.

❓ Can I print markdown files rendered by MarkView?

Yes! MarkView is print-friendly.

  1. Press Ctrl+P (or Cmd+P on Mac)
  2. The print dialog will open with all images loaded automatically
  3. Sidebars and UI elements are hidden in print view
  4. Print or save as PDF

Lazy-loaded images are automatically loaded before printing to ensure complete output.

❓ Does MarkView collect my data?

No. MarkView does NOT collect any personal data.

  • Your markdown files are never uploaded or transmitted
  • Your settings are stored locally on your device only
  • No analytics, tracking, or data collection
  • All processing happens in your browser

Read our full Privacy Policy for details.

❓ What markdown features are supported?

MarkView supports GitHub Flavored Markdown and more:

  • ✅ Headings, paragraphs, line breaks
  • ✅ Bold, italic, strikethrough
  • ✅ Lists (ordered, unordered, nested)
  • ✅ Task lists (checkboxes)
  • ✅ Links and images
  • ✅ Code blocks with syntax highlighting (180+ languages)
  • ✅ Tables
  • ✅ Blockquotes
  • ✅ Horizontal rules
  • ✅ Mermaid diagrams (flowcharts, sequence, Gantt)
  • ✅ KaTeX math equations
  • ✅ Emoji :smile:
  • ✅ HTML (rendered as-is)
